# 📌 A decisão mais importante do projeto inteiro

Quero propor que **o jogo seja orientado a eventos** (*event-driven*).

Exemplo de uma sequência:

```
Evento:
Corinthians vende Yuri Alberto

↓

Motor cria evento:
Entrada de R$ 110 milhões

↓

Motor cria evento:
Ataque perdeu qualidade

↓

Motor cria evento:
Torcida insatisfeita

↓

Motor cria evento:
Venda de camisas diminui

↓

Motor cria evento:
Patrocinador revisa expectativa

↓

Motor cria evento:
Treinador pede reposição

↓

Motor cria evento:
Scout inicia busca por atacante
```

Percebe a diferença?

Em vez de cada sistema conversar diretamente com todos os outros, **todos escutam os eventos do mundo**.

Essa arquitetura não é apenas melhor para programar. Ela também torna muito mais fácil adicionar novas funcionalidades no futuro.

Se daqui dois anos você quiser criar um sistema de agentes FIFA, redes sociais, investidores árabes ou até eleições da federação, basta criar novos eventos e novos "ouvintes". O restante do jogo continua funcionando.
