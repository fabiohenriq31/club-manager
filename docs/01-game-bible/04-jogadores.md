# A ideia que pode mudar completamente o mercado

Agora vem uma ideia que eu nunca vi em outro manager.

## Cada jogador terá uma **Reputação Percebida**.

Não apenas "qualidade".

Exemplo.

Dois atacantes.

Os dois fazem.

20 gols.

---

Primeiro.

Joga no Real Madrid.

Seleção.

Champions.

Marca forte.

---

Segundo.

Joga no Ceará.

Nunca saiu do Brasil.

---

Mesmo rendimento.

Mercado completamente diferente.

Porque existe percepção.

Isso influencia:

- valor de mercado;
- patrocinadores;
- venda de camisas;
- interesse de outros clubes;
- convocação;
- imprensa.

Ou seja, **o contexto importa tanto quanto os números**.

# O conceito mais importante do documento

Quero que os jogadores sejam tratados como **ativos vivos**.

Eles envelhecem.

Mudam de ideia.

Criam amizades.

Mudam de empresário.

Casam.

Têm filhos.

Aprendem idiomas.

Mudam de país.

Mudam objetivos.

Mudam personalidade.

E, principalmente...

**Eles lembram como foram tratados.**

Se você bancou um jogador durante uma lesão de oito meses, ele pode aceitar renovar por menos. Se você prometeu espaço e não cumpriu, talvez ele nunca mais confie em você — mesmo que ofereça um salário maior.

# 📌 Uma proposta de arquitetura (que vai facilitar MUITO o desenvolvimento)

Eu não faria um objeto `Player` gigantesco com centenas de campos.

Eu dividiria em módulos:

```
Player
│
├── Dados
├── Atributos Técnicos
├── Atributos Mentais
├── Personalidade
├── Saúde
├── Contrato
├── Mercado
├── Histórico
├── Relacionamentos
└── Objetivos
```

Essa separação deixa o banco de dados mais organizado, facilita manutenção e permite adicionar novos sistemas (como psicologia ou redes sociais) sem precisar remodelar tudo.
